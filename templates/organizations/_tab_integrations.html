Integration Instructions

Step 1 — Configure Wazuh Webhook

Open your Wazuh Manager ossec.conf and add the following integration block:

<integration>
  <name>custom-webhook</name>
  <hook_url>https://api.aegisphere.com/api/v1/ingest</hook_url>
  <level>3</level>
  <jsonout>yes</jsonout>
</integration>

Step 2 — Add API Key Authentication

Modify the integration script to include your AegiSphere API key in the headers. Generate an API Key from the Access tab.

Authorization: ApiKey <YOUR_API_KEY>

Step 3 — Restart Wazuh Manager

Apply the changes by restarting the Wazuh Manager service:

systemctl restart wazuh-manager

Alternatively: Send Logs via cURL

You can manually test ingestion by sending a JSON payload via cURL:

curl -X POST https://api.aegisphere.com/api/v1/ingest \
  -H "Authorization: ApiKey YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
        "source": "wazuh",
        "events": [{
             "rule": {"level": 5, "description": "SSH Authentication failed"},
             "agent": {"name": "server-01"}
        }]
      }'
